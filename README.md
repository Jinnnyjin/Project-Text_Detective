# [Project]Text_Detective

[기획서] 명탐정 키우기: 시스템 상세 설계 
1. 프로젝트 개요 
 - 목표: 텍스트 기반의 성장형 추리 시뮬레이션 게임. 
 - 핵심 루프: [수사/성장/자원관리] → [정보 80% 확보 시 '추리하기' 활성화] → [법정 배틀]
   
2. 시스템 상세 설계 
 - 자원 및 행동 시스템 
   턴 & 체력 관리: 모든 행동은 1턴 소모. 체력(Stamina)이 0이 되면 강제 귀가(턴 +3 페널티). 

   행동 목록: 
      조사(Search): 정보 획득 (장소 무관) 
      이동(Move): 장소 변경 (장소 무관) 
      운동(Exercise): 최대 체력(MaxStamina) 영구 증가 (집에서만) 
      공부(Study): 관찰력/판단력/추리력 상승 (집에서만) 
 
 - 스탯 및 수사 로직 
   관찰력: 수사 시 '더미 정보' 배제, 핵심 단서 우선 획득 확률(턴 효율 향상). 
          관찰력이 높으면 단번에 핵심 단서를 찾아 '쓸데없는 조사로 턴을 낭비하는 일'을 줄여줍니다. 
   판단력: 수집 정보의 진위 여부(가짜 증거) 판별 및 필터링. 
          판단력이 높으면 "이 아이템은 범인과 무관함" 혹은 "이 정보는 확실함"처럼 시스템이 확실한 힌트를 줍니다.
    
   추리력: 사건 로그의 가독성 향상. 레벨업 시 로그창 출력 범위(Line) 단계적 확장. 

  - 추리(법정 배틀) 시스템 
   활성화 조건: 총 정보량의 80% 이상 수집 시 행동 활성화. 
   중요 단서 로직: 80%를 모았더라도 '필수 핵심 증거' 누락 시 최종 단계에서 실패 판정. 

  법정 배틀 프로세스: 
    1. 준비: 총 5개의 변론 데이터 중 3개를 무작위 선택하여 제시. 
    2. 반박: 플레이어는 대응하는 증거/증언을 제시. 3단계 연속 성공 시 최종 승리. 
    3. 패배 페널티: 선택 실패 시 즉시 추리 종료 및 턴 +5 페널티 부여.
